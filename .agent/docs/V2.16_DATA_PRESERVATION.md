# V2.16 — Runtime purge and data-preservation contract

## Status

This document is normative for V2.16 runtime reset, relocation and legacy-output migration.

It complements:

- `.agent/docs/TENOR_INIT_SINGLE_AUTHORITY.md`;
- `.agent/rules/tenor-init-v2.json`;
- `.agent/skills/init-tenor/SKILL.md`;
- `.agent/docs/DOCUMENTATION_SYNC_POLICY.md`.

## Terrain defect that forced this contract

A real `TENOR_INIT_LEGACY_INSTALLATION` exposed the following destructive ordering:

```text
prepare_tenor_init
→ ensure_fresh_installation_state
→ purge_project_bound_state
→ shutil.rmtree(.agent/state)
→ canonical .agent/state/outputs/scribe-out deleted
→ migrate_all_legacy_outputs
→ legacy content moved into the now-empty canonical location
```

The observable consequence was that a canonical file containing `canonical` became `legacy` even though the migration layer itself correctly implemented “canonical destination wins”. The purge had deleted the destination before migration could enforce that rule.

This is a runtime-purge defect, not a harmless test mismatch.

## Canonical separation

`.agent/state/` contains two different classes of information:

### Project-bound coordination state — resettable

The following surfaces are bound to the old process/session/project runtime and may be removed during a proven relocation, legacy reset or corrupt-installation recovery:

```text
.agent/state/runtime/
.agent/state/proof/
.agent/state/locks/
.agent/state/sessions/
.agent/state/agents/
.agent/state/redteam/
.agent/state/backups/
.agent/state/install/
```

This includes SQLite coordination files, WAL/SHM files, stale proofs, active claims, leases, locks and installation manifests.

### Canonical/generated evidence — preservable

The following surface is not deleted by runtime purge:

```text
.agent/state/outputs/
```

It may contain:

```text
.agent/state/outputs/scribe-out/
.agent/state/outputs/graphify-out/
```

The preservation rule is byte-for-byte and recursive, including nested output files.

## Required purge behavior

`purge_project_bound_state()` must:

1. validate that the target is exactly `<project_root>/.agent/state`;
2. reject a symlinked state directory;
3. validate every preserved surface before deleting any runtime child;
4. reject `outputs` when it is a symlink or not a real directory;
5. preserve `.agent/state/outputs/` in place;
6. remove all other project-bound state children;
7. recreate `.agent/state/install/`;
8. return the preserved paths in its machine report;
9. remain idempotent and safe to retry after interruption.

A refusal must return:

```text
PROJECT_BOUND_STATE_PURGE_REFUSED
```

An unsafe preserved path must fail closed before partial purge.

## Migration conflict policy

After runtime reset, legacy root outputs may still need migration:

```text
scribe-out/
graphify-out/
```

The conflict policy is mandatory:

```text
canonical destination absent
→ move legacy item into canonical destination

canonical destination present
→ canonical item wins
→ move legacy item into:
   <canonical>/_legacy_migrated/<UTC timestamp>/
```

No canonical content may be overwritten silently. No legacy content may be discarded silently.

## Graphify safety after preservation

Preservation is not readiness.

A preserved Graphify output may have been copied from another root or generated for an older workspace fingerprint. TENOR INIT must therefore run the existing Graphify readiness checks after purge and migration.

`TENOR_INIT_READY` remains forbidden unless Graphify proves:

- project-root match;
- workspace-fingerprint match;
- supported `nodes + links` or historical `nodes + edges` schema;
- no smoke fixture on terrain;
- no stale, partial, wrong-root or contradictory graph.

A preserved but invalid graph is retained for forensic/no-loss purposes, but it cannot authorize writes.

## SAME_PROJECT interaction

`TENOR_INIT_SAME_PROJECT` is a different invariant:

- no forced installer;
- no tracked-file rewrite;
- no runtime purge;
- only bounded session/runtime mutations;
- drift reported read-only;
- repair explicit through `scribe install --force`.

The output-preservation contract applies when a purge is actually required. It does not make `SAME_PROJECT` destructive.

## Executable proof

The contract is enforced by:

```text
.agent/mcp/tests/test_installation_output_preservation.py
.agent/workflow/scribe/sel/tests/test_tenor_init_output_hygiene.py
.agent/workflow/scribe/sel/tests/test_generated_surface_parity.py
.agent/scripts/validation_suite.py
.github/workflows/v216-portability.yml
```

The acceptance criteria are:

```text
runtime/proof/locks removed
nested canonical outputs unchanged byte-for-byte
legacy conflicts quarantined
unsafe output symlink refused before deletion
Graphify revalidated before readiness
generator/surface parity exact
zero red regression test
```
